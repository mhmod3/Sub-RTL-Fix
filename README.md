
# السلام عليكم

هذا مشروع بسيط، وظيفته الأساسية إصلاح مشكلة شائعة في ملفات الترجمة: عدم دعم اتجاه الكتابة من اليمين إلى اليسار (RTL) عند مزج النص العربي مع أرقام أو كلمات إنجليزية أو علامات ترقيم.

الموقع يتعامل مع ثلاث صيغ رئيسية: **SRT** و **VTT** و **ASS**، ويقوم بمعالجتها جميعاً بنفس الطريقة الذكية.

---

## مثال على ملف SRT

**قبل المعالجة (النص مشوش):**
```
1
00:00:05,000 --> 00:00:10,000
‫3 مرحبا بك في الدرس رقم‬
‫‬
2
00:00:12,000 --> 00:00:18,000
هذا نص مختلط مع كلمة Hello ورقم 2024
```

**بعد المعالجة (يظهر بالاتجاه الصحيح):**
```
1
00:00:05,000 --> 00:00:10,000
مرحبا بك في الدرس رقم 3

2
00:00:12,000 --> 00:00:18,000
‫هذا نص مختلط مع كلمة Hello ورقم 2024‬
```

0

## مثال على ملف VTT

```
في هذا الفيديو سنتعلم عن CSS

استخدام flexbox مع direction: rtl
```

## مثال على ملف ASS

**قبل المعالجة:**
```
[Script Info]
Title: ترجمة عربية

[Events]
Format: Layer, Start, End, Style, Name, MarginL, MarginR, MarginV, Effect, Text
Dialogue: 0,0:00:06.00,0:00:10.00,Default,,0,0,0,,رقم 100 وكلمة Right
```

**بعد المعالجة:**
```
[Script Info]
Title: ترجمة عربية

[Events]
Format: Layer, Start, End, Style, Name, MarginL, MarginR, MarginV, Effect, Text
Dialogue: 0,0:00:06.00,0:00:10.00,Default,,0,0,0,,‫رقم 100 وكلمة Right‬
```

## كيف يعمل الموقع باختصار؟

- ترفع ملف الترجمة (اسحبه أو اختره).
- الموقع يكتشف ترميز الملف تلقائياً (UTF-8, Windows-1256, ...).
- يحدد الصيغة بنفسه (SRT, VTT, ASS).
- يطبق الإصلاح على النصوص فقط، تاركاً التوقيتات والأرقام والوسوم كما هي.
- النتيجة تظهر لك فوراً، ويمكنك تحميل الملف المعدل أو نسخ النص.
